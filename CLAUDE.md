## 작업 규칙

### 1. 승인 원칙
- "시작" 또는 "go"라는 말이 나오기 전까지는 코드를 읽고 분석만 하고, 실제로 파일을 만들거나 수정하지 않을 것.
- 여기서 "단계"는 1~9단계 같은 큰 번호 순서가 아니라 **하나의 완결된 기능 단위**(예: 헤더, 슬라이더, 더보기 메뉴의 "내보내기" 기능 등)를 뜻함.
  - 기능 하나가 UI(예: 3단계)와 실제 동작 로직(예: 5단계)처럼 서로 다른 번호의 체크리스트에 걸쳐 있을 수 있음. 이런 경우 UI만 먼저 만들고 로직은 나중에 따로 만들지 않고, **기능이 실제로 동작하도록 관련 항목을 한 번에 묶어서** 진행하는 것을 기본으로 함 (재작업 비용을 줄이기 위함).
  - 다만 서로 다른 기능을 한 세션에 여러 개 묶어 진행하는 것은 여전히 금지 (절대 금지 사항 참고).
- 승인은 **세션 전체에 한 번이 아니라, 기능 단위마다 매번** 받을 것.
- 한 기능이 끝나면 자동으로 다음 기능으로 넘어가지 말고, 반드시 새로 "시작"/"go"를 기다릴 것.
- 작업을 시작하기 전, 다음 형식으로 계획을 먼저 설명할 것.
  > "이번 단계에서는 [무엇을] 위해 [어떤 파일들]을 [어떻게] 수정하겠습니다."
  - 하나의 기능이 여러 번호의 체크리스트 항목을 함께 완료시키는 경우, 계획 설명 시 어느 항목들이 함께 체크되는지 명시할 것.

### 2. 작업 기록 규칙 (이전 작업 기억하기)
- 새 세션을 시작할 때 먼저 `PROGRESS.md` 파일이 있는지 확인하고, 있으면 내용을 읽을 것.
- 가장 마지막에 기록된 작업 내용을 요약해서 사용자에게 보고할 것.
- "다음 작업 제안"이 적혀 있으면 함께 보고할 것.
- `PROGRESS.md`가 없으면 "이전 작업 기록이 없습니다"라고 알릴 것.
- `PROGRESS.md` 맨 아래에 기록을 추가할 것 (기존 기록은 지우지 말고 누적할 것).
- 작업 순서는 체크리스트를 확인하여 순서대로 진행하되, 번호 순서는 권장 순서일 뿐 절대 규칙은 아님 — 진행 중인 기능이 다른 번호의 항목과 연결되어 있다면 그 항목까지 함께 진행 가능 (위 "1. 승인 원칙" 참고)
- 진행 완료된 체크리스트에 체크 표시 할 것
- 작업 전 항상 계획을 먼저 설명하고 승인을 받을 것
- 파일 수정 시 변경 내용을 먼저 요약할 것
- 각 작업 완료 후 CLAUDE.md 체크리스트를 반드시 업데이트 할 것
- [x] 처리 시 완료 날짜를 괄호로 표기할 것. 예: - [x] 로그인 UI (2026-04-01)

### 3. 검증 규칙 (기능 구현 후 확인)
- 자동화 테스트(로컬 서버/헤드리스 브라우저 등, `localhost`·`file://` 기준)를 통과했다고 해서 실제 환경에서도 정상 동작한다고 단정하지 말 것.
- 기능 구현 완료 후 확인은 2단계로 나눠 진행할 것.
  1. Claude가 자동화 테스트로 1차 확인
  2. 사용자가 실제 기기/네트워크 환경(폰 브라우저, 필요 시 PC를 LAN IP로 접속 등)에서 2차 확인
- 2차 확인이 끝나기 전까지는 다음 기능 단계로 넘어가지 말 것.

## 메모 데이터 구조

모든 메모는 아래 구조로 저장한다. **2단계부터 이 구조를 그대로 사용할 것.**

```json
{
  "id": "uuid",
  "content": "메모 내용",
  "folder": null,
  "starred": false,
  "images": [],
  "createdAt": "2026-04-01T00:00:00.000Z",
  "updatedAt": "2026-04-01T00:00:00.000Z"
}
```

| 필드 | 타입 | 설명 |
|------|------|------|
| `id` | string | crypto.randomUUID() 로 생성 |
| `content` | string | 메모 본문 전체 |
| `folder` | string \| null | 폴더명 (없으면 null = 미분류) |
| `starred` | boolean | 중요 메모 여부 |
| `images` | array | Base64 이미지 배열 (초기값 []) |
| `createdAt` | ISO8601 | 생성 시각 |
| `updatedAt` | ISO8601 | 마지막 수정 시각 |
